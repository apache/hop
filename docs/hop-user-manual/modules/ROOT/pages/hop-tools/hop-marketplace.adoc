:description: Install optional Hop plugins from Apache Repository / Maven Central using the marketplace CLI or GUI.

= Hop Marketplace

Some Hop plugins are optional: they are not bundled in the default client zip so the download stays under the ASF packaging size limit. The *marketplace* installs those plugins into the running Hop client’s `plugins/` tree from a Maven repository.

== Defaults

Without extra configuration, Hop uses:

. *asf* (primary) — `https://repository.apache.org/content/groups/public/`
. *central* (fallback) — `https://repo1.maven.org/maven2/`

Install tries the primary repository first, then other *enabled* repositories, until the plugin zip is found.

Settings are stored under the `marketplace` key in `${HOP_CONFIG_FOLDER}/hop-config.json` (always that file).

== CLI

From the Hop install directory:

[source,bash]
----
./hop marketplace install hop-tech-parquet
./hop marketplace install hop-engines-spark:2.19.0
./hop marketplace list
./hop marketplace uninstall hop-tech-parquet

# Repositories
./hop marketplace repo list
./hop marketplace repo add --id corporate --url https://nexus.example.com/repository/hop/ --primary
./hop marketplace repo set-primary asf
./hop marketplace repo reset-defaults

# Declarative environment (CI / Docker)
./hop marketplace apply -f hop-env.yaml
./hop marketplace validate -f hop-env.yaml
----

After install or uninstall, *restart Hop* so the plugin registry reloads.

Coordinates: `artifactId`, `artifactId:version`, or `groupId:artifactId:version`.

== GUI

*Tools → Marketplace…* lists featured optional plugins, installs/uninstalls them, and opens *Manage…* to edit repositories (same config as the CLI).

== hop-env.yaml

For reproducible environments (Docker, CI):

[source,yaml]
----
version: "1.0"
hopVersion: "2.19.0"
repositories:
  - id: asf
    url: "https://repository.apache.org/content/groups/public/"
plugins:
  - artifactId: hop-tech-parquet
    version: "2.19.0"
----

See also the sample under `config/projects/samples` after the marketplace plugin is present, or the source sample in `plugins/misc/marketplace`.

== Enterprise / air-gapped

Point marketplace repositories at your internal Nexus or Artifactory mirror of `org.apache.hop` plugin zips. Prefer anonymous read where possible; for private repos set Basic auth on the repository entry or use `HOP_MARKETPLACE_USERNAME` / `HOP_MARKETPLACE_PASSWORD`. Wrong credentials can cause HTTP 401 even when anonymous access would work.
